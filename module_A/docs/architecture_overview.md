## 1. Выбор архитектурного подхода: медальонная (Bronze–Silver–Gold)

Обоснование:
- Простота реализации и понимания – критично для регионального чемпионата с ограниченным временем.
- Хорошо сочетается с ClickHouse и dbt.
- Позволяет хранить историю через SCD2 в Silver слое.
- Легко расширяется: новый источник данных кладётся в Bronze, затем трансформируется в Silver.

Слои:
- Bronze – сырые данные, как есть (видеокадры, JSON-логи, внешние API). Неизменяемые, хранятся в MinIO.
- Silver – очищенные, нормализованные, валидированные данные с поддержкой истории изменений (SCD2). Хранятся в ClickHouse.
- Gold – агрегированные витрины данных, оптимизированные для BI и ML. Хранятся в ClickHouse.

## 2. Логические слои и технологии

Слой | Назначение | Технология 
   - Bronze | Сырые данные с камер, погода, ДТП | MinIO (S3) + Parquet 
   - Silver | Очищенные детекции, справочники с историей | ClickHouse (ReplacingMergeTree для SCD2) 
   - Gold | Витрины (загруженность, прогнозы, структура потока) | ClickHouse (SummingMergeTree, звёздные схемы) 
   - Потоковая обработка | Из камер -> Kafka -> Bronze | Kafka + Spark Streaming 
   - Пакетная обработка | Bronze -> Silver -> Gold | Airflow + dbt 


## 3. Технологический стек 

Компонент | Выбор | Обоснование 
   - Хранилище данных | ClickHouse | Колоночная СУБД, отличная производительность на агрегациях, поддержка TTL, SCD2 через ReplacingMergeTree. 
   - Озеро данных | MinIO + Parquet | S3-совместимое, дешёвое масштабируемое хранение для слоя Bronze. Parquet – эффективное сжатие. 
   - Оркестрация | Apache Airflow | Управление DAG-пайплайнами, инкрементальная загрузка Bronze→Silver→Gold. 
   - Потоковая обработка | Kafka + Spark Streaming | Приём видеодетекций в реальном времени, запись в Bronze. 
   - Трансформации | dbt (data build tool) | Версионирование SQL-трансформаций Silver→Gold, тестирование качества, документация. 
   - BI дашборды | Apache Superset | Интерактивные дашборды, поддержка ClickHouse, open-source. 
   - Прогнозный движок | MLflow + Prophet/XGBoost | Обучение, регистрация моделей, инференс по расписанию (читает Gold, пишет в Gold). 

## 4. Архитектурная диаграмма потоков данных
(/architecture_flow.drawio)
Краткое описание:
1. Видеопоток -> детекция (нейросеть) -> Kafka -> Spark Streaming -> Bronze (MinIO).
2. Пакетный пайплайн Airflow + dbt:
   - Читает Bronze -> очистка/нормализация -> запись в Silver (ClickHouse, SCD2).
   - Из Silver -> агрегация -> Gold витрины (ClickHouse).
3. Дашборд Superset читает Gold.
